# Roadmap

## Fase 1: MVP Desktop Offline

Objetivo: construir el nucleo de lectura y estudio local.

- Crear proyecto Flutter base.
- Definir arquitectura por capas.
- Crear modelos de Biblia, version, libro, capitulo y versiculo.
- Implementar SQLite local.
- Importar versiones iniciales: Reina-Valera y TLA.
- Crear Reader Controller.
- Implementar buffer de capitulos y versiculos.
- Crear modo lectura enfocada.
- Crear modo capitulo completo.
- Crear busqueda simple.
- Crear notas.
- Crear resaltados.
- Crear favoritos y marcadores.
- Crear historial.
- Crear doble panel.
- Crear modo presentacion.
- Crear sistema centralizado de shortcuts.

## Fase 2: Experiencia Pro

- Temas claro, oscuro y base para temas personalizados.
- Configuracion de fuentes y tamano.
- Selector rapido de versiones.
- Exportacion basica.
- Mejoras de UX.
- Refinamiento de animaciones.

## Fase 3: Backend

- FastAPI.
- PostgreSQL.
- Autenticacion.
- Cuentas.
- Gestion de dispositivos.
- Sincronizacion.
- WebSockets.
- Descarga dinamica de versiones.

## Fase 4: Mobile

- Lectura movil rapida.
- Notas y resaltados tactiles.
- Sincronizacion con Desktop.
- Control remoto de presentacion.
- Conexion por QR y sesiones temporales.

## Fase 5: Web Completa

- Landing page.
- Portal de descargas.
- Cuentas.
- Documentacion.
- Lector web futuro.
- Estudios.
- Comunidad.
- Administracion.
